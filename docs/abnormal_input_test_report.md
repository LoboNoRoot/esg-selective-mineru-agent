# 异常输入处理测试报告

## 测试目标

验证系统在遇到不适合 ESG 60 字段抽取流程的报告时，能够主动跳过并给出可追溯原因，而不是继续执行 MinerU/LLM 抽取并产生低可信结果。

## 测试样例

- 文件：`000690_宝新能源_2024_宝新能源：2024年度社会责任报告.pdf`
- 类型判断：传统社会责任报告
- 首页标题：`广东宝丽华新能源股份有限公司 2024 年度社会责任报告`
- 参照标准：`深圳证券交易所《上市公司社会责任指引》`

## 跳过规则

系统读取文件名和 PDF 前两页文本，若命中以下条件，则标记为 `skipped`：

- 标题或首页命中 `社会责任报告` / `企业社会责任报告`
- 同一标题窗口中未出现 `ESG`、`环境、社会和公司治理`、`环境、社会及治理`、`可持续发展` 等 ESG 披露框架信号

该样例命中：

```text
reason_code = legacy_social_responsibility_report
```

## 预期行为

- API job 状态：`skipped`
- 不进入 LLM 字段抽取
- 不生成伪装成正常结果的 `extraction_results.json`
- 输出 `skip_report.json`
- 前端显示“已跳过”和跳过原因

## 价值说明

该处理展示了算法在不同输入条件下的边界能力：

- 对标准 ESG / 可持续发展 / 环境社会治理报告继续执行抽取
- 对传统社会责任报告、摘要、鉴证声明等非目标正文报告主动跳过
- 避免异常输入污染评估集和抽取准确率统计

## 测试结论

系统已具备异常输入识别和跳过能力。对于本样例，推荐结论为：跳过，不纳入 ESG 60 字段抽取结果集。
